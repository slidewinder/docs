Basics

  1. Node.js and coffeescript
  2. Setting up for slidewinder development

Code style

We enforce a coffeescript style guide - make sure you read it!.

The main slidewinder repo has an .editorconfig file. This allows your editor to automatically adopt many of our style conventions when you're working on files in that repo. This is especially useful if you often switch between projects.

See how to set up .editorconfig with your text editor at the editorconfig site.

What kinds of help do we want?

We have a roadmap for the project, which is managed using Github Issues and milestones.

You can browse the issues by repo. Or if you look at the milestones for a repo, you can browse the issues by milestone.

Report bugs

If you find a bug, please let us know by creating a new issue. If you just have a question, please use the chat room instead of filing a new issue.

Fixing bugs / implementing features

Any issue that has a help wanted label is up for grabs! Check out the list of all those issues.

In addition, any issue with the good first issue label is one the developers think would make a good starting point for a new contributor. You can list those too.

Suggesting features or changes

Think we're missing a feature? Or perhaps we could be doing something better? We want to hear about it!

Please file an issue to discuss proposed changes before making a pull request. We respect your time and would like to maximise the chance of your contributing being used. You can also drop by the chat room to talk about potential improvements.

Not sure how to get started?

We've got a live chat room where you can come and ask us questions - we'd love to see you there, and the team will be happy to help you find your first issue.
